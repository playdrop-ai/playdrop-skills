# Platform Capabilities

Use this reference when the task is about whether a concept or project fits Playdrop at all.

## Core capability groups

- web-hosted games and interactive apps
- public listing and discovery surfaces
- creator-facing CLI workflows for develop, validate, capture, and publish
- player identity, avatars, saves, and multiplayer services
- catalog assets and generated assets for visuals and audio
- monetization, boosts, and distribution support where public docs say they apply

## Planning guidance

- choose only the platform features that materially improve the project
- do not force platform services into a concept that does not need them
- use this as a fit check during planning, upgrades, and marketing positioning

## Messaging guidance

- describe Playdrop in terms of concrete creator outcomes
- match claims to the live public docs and current shipped workflows
- avoid vague “all-in-one” framing when a more specific capability is available
