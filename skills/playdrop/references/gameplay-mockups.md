# Gameplay Mockups

Use this when creating or reviewing gameplay mockups before implementation.

## Goal

Make mockups that act as literal build targets, not vibe boards.

## Rules

- start from the simplified plan, not the full wish list
- mock the best platform first
- create one mockup per key screen and supported surface that actually survived scope cutting
- show the real framing, HUD, interaction model, and gameplay hierarchy
- keep the play surface edge to edge when the concept allows it
- if touch is primary, show the actual control affordance for any frequent action
- avoid explanation arrows, labels, and text that would not ship in the product
- review exported mockups at shipping resolutions before treating them as implementation-ready
- if the mockup cannot guide implementation directly, it is not ready

## Review focus

- framing and camera distance
- HUD placement and density
- control clarity
- danger, reward, and fail-state readability
- small-screen legibility
- art direction consistency
