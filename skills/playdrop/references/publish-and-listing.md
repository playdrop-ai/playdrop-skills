# Publish And Listing

Use this when preparing a project for discovery, proof, and release.

Start with:
- `playdrop project validate .`
- `playdrop project capture`
- `playdrop project capture remote`
- `playdrop project publish .`

Suggested order:
- validate the project
- capture gameplay proof
- capture screenshots and listing media
- publish when the listing and build are ready

Typical listing needs:
- icon
- hero
- screenshots
- short description
- genre and tags
