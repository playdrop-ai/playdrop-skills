# Migration Guide

Use this when updating CLI, plugin, or SDK usage.

## Checklist

- check current CLI version with `playdrop --version`
- read the relevant changelog or live docs
- update the project only as far as needed
- validate with `playdrop project validate .`
- run the dev flow after version-sensitive changes

## Priority

Fix compatibility and broken workflows first. Nice-to-have upgrades come after the project is stable again.
