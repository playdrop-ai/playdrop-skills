# Discovery

Use this when the user wants inspiration, references, assets, demos, or faster implementation paths.

Start with:
- `playdrop browse ...`
- `playdrop search <query>`
- `playdrop detail <ref> --json`
- `playdrop documentation browse`
- `playdrop browse --kind app --app-type template --json`

Workflow:
- inspect 2 to 3 strong references before changing art, controls, systems, or packaging
- prefer the closest published starter, demo, or game over inventing structure from scratch
- look across apps, assets, and packs before deciding

Decision rule:
- use a reference as inspiration when only the vibe matches
- align with a reference when the genre or UX matches
- remix when the public project is already close to the target
