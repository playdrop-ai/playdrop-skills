# Engine Porting

Use this skill when the project starts outside the normal Playdrop starter flow.

## Workflow

1. identify the closest Playdrop starter or public reference
2. use starter kits as the default migration baseline unless there is a better fit
3. freeze source scope before rewriting systems
4. validate core loop, controls, and performance early
5. only then move to listing or publish work

- `godot-porting.md`
