# Scope Control

Use this skill after an initial concept or spec exists but before heavy implementation starts.

## Workflow

1. Restate the core promise in one sentence
2. Identify the one mechanic or interaction that must survive
3. Cut platforms, modes, meta systems, screens, and asset work until the v1 is obviously finishable
4. Write a clear `not in v1` list
5. Decide whether the simplified concept is still worth building or should be reshaped

## Rules

- prefer one strong promise over several weak ones
- if simplification removes the differentiator, stop and send the concept back to planning
- do not keep features just because they sound good in prose
- the result must still support a real session, not a 30-second toy

## Outputs

- simplified v1 summary
- core mechanic
- `not in v1` list
- continue or rethink recommendation
