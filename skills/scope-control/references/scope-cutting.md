# Scope Cutting

Use this when a game plan is still too broad and needs to become a strong v1.

## Goal

Reduce the concept to the smallest strong version worth shipping.

## Rules

- prefer one strong promise over several weak ones
- keep the one mechanic or interaction that makes the game worth opening
- cut extra platforms, modes, progression, currencies, achievements, or multiplayer unless they are core
- cut screens, popups, tutorials, and explanatory UI that exist only to rescue a muddy idea
- write an explicit `not in v1` list instead of keeping fuzzy future scope
- if simplification removes the differentiator or reveals a thin clone, reshape the concept before implementation
- the simplified version still needs a real session and a reason to replay

## Output

Produce:

- one-sentence core promise
- the mechanic that must survive
- the final v1 scope
- the `not in v1` list
- a continue or rethink recommendation
