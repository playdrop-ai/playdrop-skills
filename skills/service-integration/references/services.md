# Services

Use this when the project needs online or platform-backed features.

Service areas:
- identity and avatars for player profile and sign-in
- saves for persistence across sessions or devices
- realtime multiplayer for shared live gameplay
- monetization for IAP or paid content
- AI generation for fast asset or content creation

Workflow:
- read public docs first
- integrate only the services that materially help the project
- prefer the smallest useful service set over platform overreach

Suggested path:
- identify which service the game actually needs
- start from `playdrop documentation browse`
- inspect docs and current CLI help for that service area
- look for a published demo or reference before implementing
