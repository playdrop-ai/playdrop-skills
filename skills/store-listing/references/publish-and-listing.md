# Publish And Listing

Use this when preparing a project for discovery, proof, and release.

Start with:
- `playdrop project validate .`
- `playdrop project capture`
- `playdrop project capture remote`
- `playdrop project publish .`

Suggested order:
- validate the project
- capture the strongest real gameplay moments first
- pick screenshots from the real build before generating marketing art
- generate or compose hero and icon assets from gameplay references
- write short copy that explains how to play and what matters
- review the full listing family before publish
- publish when the listing and build are ready

Typical listing needs:
- icon
- portrait and landscape hero when needed
- screenshots
- gameplay video
- short description
- genre and tags

Rules:
- do not use a raw crop of hero art, screenshots, or other rectangular listing images as the final icon
- prefer existing dedicated icon art when the project already has one
- otherwise generate a purpose-built square icon from the hero art and gameplay references so it reads clearly at small sizes
- hero and icon should be purpose-built marketing assets that still match the shipped art direction
- listing art should not promise a stronger fantasy or cleaner build than the runtime actually delivers
- screenshots should show real gameplay moments, not menus or loading states
- include a real gameplay video before treating the release as ready
- if the rules could be misread, say exactly what counts as a match, clear, score, or fail
- review the icon, hero, screenshots, and copy together so the package feels coherent
