# Existing Projects

Use this when the user already has a local project and wants to make it Playdrop-native.

Start with:
- `playdrop project init .`
- `playdrop project validate .`

Check:
- project metadata, typically `catalogue.json`
- whether the project should stay in place or be remixed from a closer starter
- whether assets, services, or listing work are also in scope

Default assumption:
- if the user says use `$playdrop`, registration and reference research are in scope unless they narrow the request
